js面向对象

js面向对象

在js中不存在类这个东西，而只有对象

对象：无序属性的集合，属性可以包括基本值，对象和函数

属性类型

js对象中有两种属性

数据属性

数据属性有一个能够描述值的位置，有四个描述行为的特性

属性 作用
Configurable 能否修改属性的特性，能否删除该属性
Enumerable 能否通过for in遍历属性
Writable 能否对属性进行写操作
Value 存储属性的值

默认情况下前面三个特性都被设定为True

如果要修改属性的特性值，则只能使用Object.defineProperty()方法

如果对于以上方法不提供第三个参数，则会全部设置为false

访问器属性

访问器属性不包含值，而是有一对getter和setter函数，用来控制对于数据属性的访问

访问器有四个特性

特性 作用
Configurable 能否修改属性的特性，能否删除该属性
Enumerable 能否通过for in遍历属性
Get 能否读取属性
Set 能否写属性

访问器属性的特性设置同样需要Object.defineProperty()方法

这里实际上是通过一个year的访问器属性来对属性_year进行访问，可以更加有效的管理属性的访问

声明多个属性

如果要声明多个属性则可以使用Object.defineProperties()方法

读取属性的特性

可以通过Object.getOwnPropertyDescriptor()来获取一个属性的特性值

这样对于一个对象的任意一个属性都可以直接获取其特性

创建对象

工厂模式

是一种设计模式，指通过一个返回对象的函数进行批量的实例化

但是工厂模式有一个问题，就是无法确定对象的种类

构造函数模式

指通过构造函数，使用new进行实例的创建

使用构造函数模式创造的实例可以用instanceof判断是哪一种对象

但是如果使用构造函数模式，则其中的每一个函数都要重新构造一次。因此可以采取以下方式

原型模式

每个函数都有一个prototype属性，这个属性可以通过以下方式使用

原型链

在上述原型链中，b、c为Foo的实例，__proto___属性指向其prototype

Foo为Foo的构造函数，Function.prototype为函数的原型，Object.prototype为所有对象的原型

注意以下情况

可以使用hasOwnProperty()方法判断属性来自实例还是来自原型

使用in操作符可以知道属性在实例中是否存在，不区分来自原型还是来自实例

如果需要重写原型对象则可以直接对prototype进行改动

但是原型方法有一个很重要的缺点，如果属性包含引用类型值，那么对于一个实例的该属性修改会导致所有实例的该属性都发生变化

构造函数与原型模式组合

最常用的方法是将构造函数与原型模式组合起来

也就是数据属性都由构造函数进行构造，方法都由原型函数进行构造

稳妥构造函数

通过这种方法能够构造不能直接访问属性的对象

使用这种方式的构造函数，由于没有数值属性（因为通过o进行声明），所以只能通过getName方法访问name属性。就限制了对于属性的直接访问

继承

js的继承一般使用过原型链实现的

可以粗浅的理解为继承就是在原型上添加属性

因此表示一个类继承另一个类只需要使用

虽然js不能重载，但是如果重写实例中的方法会将父对象的方法覆盖，且不影响父对象中的方法，

还可以在内部调用父对象的构造函数
